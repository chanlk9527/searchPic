# SearchPic SaaS Task Board

本任务板用于跟踪 SearchPic 当前服务端、SaaS 能力和后续 Web 控制台的建设计划。

---

## 阶段 1：基础设施与核心骨架

目标：把服务端基础能力先搭起来，确保数据库、消息队列、检索引擎和统一返回结构可用。

- [x] 初始化 Spring Boot 工程，完成 `pom.xml` 与 `application.yml`
- [x] 编写 `docker-compose.yml`，拉起 MySQL、Kafka、Elasticsearch、MinIO 等基础组件
- [x] 建立统一返回结构 `Result<T>` 与全局异常处理 `GlobalExceptionHandler`
- [x] 建立 Flyway 数据迁移，完成租户与核心业务表初始化
- [x] 建立 Elasticsearch 文档模型 `AlertEventDocument`

---

## 阶段 2：AI 集成层

目标：打通视觉理解、文本理解和向量提取链路，为写入和检索能力提供统一 AI 抽象。

- [x] 集成阿里云 DashScope，用于图像理解与 Embedding 提取
- [x] 集成 Google Gemini，用于自然语言查询意图解析
- [x] 补齐 AI 调用的兜底、超时和容错机制

---

## 阶段 3：写入链路

目标：让外部图片事件可以进入系统，并异步完成分析与入库。

- [x] 编写 `API_REFERENCE.md`
- [x] 实现 `POST /api/v1/events/ingest`
- [x] 实现鉴权拦截与默认租户透传机制
- [x] 将写入请求投递到 Kafka
- [x] 完成 Kafka 消费、AI 分析和 Elasticsearch 入库链路

---

## 阶段 4：检索与浏览链路

目标：支持自然语言搜索、事件浏览、基础聚合统计，满足调试和演示使用。

- [x] 实现 `POST /api/v1/search/query`
- [x] 实现用户查询意图解析与时间窗口抽取
- [x] 实现基于租户过滤的 Elasticsearch 检索
- [x] 实现事件列表接口 `GET /api/v1/events`
- [x] 实现最新事件接口 `GET /api/v1/events/latest`
- [x] 实现概览统计接口 `GET /api/v1/events/stats/overview`
- [x] 实现设备统计接口 `GET /api/v1/events/stats/by-camera`
- [x] 实现租户用户与设备目录接口 `GET/POST /api/v1/admin/users`、`GET/POST /api/v1/admin/devices`

---

## 阶段 5：单租户 Web 控制台（第一版）

目标：先做一个免登录、默认进入开发租户的 Web 端，重点服务于 SaaS 运营、调试、演示和快速验证，不先做复杂权限系统。

### 产品原则

- [ ] 打开页面后直接进入默认租户 `tenant_local_dev`
- [ ] 暂不做登录、注册、角色权限、多租户切换
- [ ] 首版优先支持“可观察、可调试、可演示”，不是做完整后台管理系统
- [ ] 所有页面优先复用现有 API；缺的统计能力再按页面反推接口

### 页面 1：总览 Dashboard

目标：让使用人员一打开就知道当前租户有没有数据、最近活跃不活跃、主要数据来自哪些设备。

- [ ] 展示核心指标卡片：事件总数、设备数、用户数、最近一条事件时间
- [ ] 展示最近 24 小时 / 7 天事件趋势图
- [ ] 展示设备上传排行榜 Top N
- [ ] 展示最新事件流，支持直接点进详情
- [ ] 展示当前默认租户标识与运行环境信息

### 页面 2：事件浏览与调试页

目标：让运营或研发可以快速翻看数据，确认写入是否成功、图片内容是否合理、字段有没有异常。

- [ ] 事件列表支持分页、时间范围筛选、用户筛选、设备筛选
- [ ] 卡片或表格中显示图片缩略图、事件 ID、用户、设备、时间、实体摘要
- [ ] 支持查看单条事件详情
- [ ] 详情页展示原图、结构化字段、实体文本、时间信息
- [ ] 预留展示 AI 解析结果原文/中间结果的区域，方便后续排查模型输出

### 页面 3：自然语言搜索调试页

目标：把核心搜索能力做成一个直观可测的调试台，便于演示和验证“搜图”效果。

- [ ] 提供查询输入框，支持中文自然语言直接搜索
- [ ] 支持附加筛选条件：时间范围、设备范围
- [ ] 页面展示本次请求参数与解析后的意图结果
- [ ] 展示搜索结果列表，包含缩略图、匹配分数、时间、设备、事件 ID
- [ ] 支持查看某条结果的命中原因摘要，后续可扩展 BM25 / 向量召回解释
- [ ] 保留最近若干次调试查询历史，方便重复验证

### 页面 4：写入调试页

目标：让使用人员不用 Postman 也能直接做写入联调。

- [ ] 提供事件写入表单：`event_id`、`camera_id`、`user_id`、`timestamp`、`image_url`
- [ ] 支持一键生成测试事件 ID 和当前时间
- [ ] 支持提交后即时显示请求结果与返回状态
- [ ] 支持展示最近提交记录，方便确认是否进入处理流程
- [ ] 后续可扩展“上传图片后自动回填 URL”的便捷能力

### 页面 5：用户与设备目录页

目标：给运营和调试人员一个直观的“数据归属视图”，知道谁名下有哪些设备、各自上传了多少数据。

- [ ] 展示用户列表：姓名、邮箱、设备数、事件数、最近上传时间
- [ ] 展示设备列表：设备名、位置、所属用户、事件数、最近上传时间
- [ ] 支持新增用户
- [ ] 支持新增设备并绑定到用户
- [ ] 支持从用户视角跳转查看其设备和事件

### 页面 6：统计分析页

目标：让 SaaS 使用人员能看到“量”和“趋势”，而不仅仅是接口有没有返回。

- [ ] 按日统计事件写入量
- [ ] 按设备统计事件贡献占比
- [ ] 按用户统计活跃度
- [ ] 展示最近无数据设备，辅助排查采集异常
- [ ] 预留后续计费/配额区域，但首版不做真实计费逻辑
